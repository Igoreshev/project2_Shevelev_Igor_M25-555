## Управление таблицами

Модуль управления таблицами позволяет создавать, удалять и просматривать таблицы базы данных.
Информация о таблицах (имя таблицы и структура столбцов) хранится в файле `db_meta.json`.

### Доступные команды

- `create_table <имя_таблицы> <столбец1:тип> <столбец2:тип> ...`  
  Создаёт новую таблицу с указанными столбцами.  
  Столбец `ID:int` добавляется автоматически.

- `drop_table <имя_таблицы>`  
  Удаляет таблицу по имени.

- `list_tables`  
  Выводит список всех существующих таблиц.

- `help`  
  Показывает справочную информацию по доступным командам.

- `exit`  
  Завершает работу программы.

### Поддерживаемые типы данных

- `int`
- `str`
- `bool`

### Пример использования

```text
>>>Введите команду: create_table users name:str age:int is_active:bool
Таблица "users" успешно создана со столбцами: ID:int, name:str, age:int, is_active:bool

>>>Введите команду: list_tables
- users

>>>Введите команду: drop_table users
Таблица "users" успешно удалена.

>>>Введите команду: exit
Выход из программы.

## CRUD-операции

После создания таблиц база данных поддерживает полный набор CRUD-операций
(Create, Read, Update, Delete) для работы с данными.

### Доступные команды

#### Добавление записи (INSERT)

```text
insert into <имя_таблицы> values (<значение1>, <значение2>, ...)
insert into users values ("Sergei", 28, true)

Чтение данных (SELECT)
select from <имя_таблицы>
select from <имя_таблицы> where <столбец> = <значение>

Обновление данных (UPDATE)
update <имя_таблицы> set <столбец> = <новое_значение> where <столбец> = <значение>

Удаление данных (DELETE)
delete from <имя_таблицы> where <столбец> = <значение>

Информация о таблице
info <имя_таблицы>

## Дополнительные возможности

В проекте реализованы продвинутые механизмы повышения надежности и удобства работы:

### Обработка ошибок
Все операции с базой данных обернуты в декоратор обработки ошибок.
Это позволяет централизованно перехватывать ошибки, такие как:
- обращение к несуществующей таблице или столбцу;
- ошибки валидации типов данных;
- проблемы с файловой системой.

Пользователь получает понятные сообщения об ошибках без аварийного завершения программы.

### Подтверждение опасных операций
Для потенциально опасных операций (удаление таблиц и записей) используется
декоратор подтверждения действий.
Перед выполнением операции пользователь должен подтвердить действие:
Вы уверены, что хотите выполнить "удаление таблицы"? [y/n]:

### Логирование времени выполнения
Для операций, работающих с файловой системой (insert, select),
реализован декоратор логирования времени выполнения.
В консоль выводится время работы функции в формате:

Вы уверены, что хотите выполнить "удаление таблицы"? [y/n]:

### Кэширование запросов
Операция select использует механизм кэширования на основе замыканий.
Результаты одинаковых запросов сохраняются и повторно используются,
что ускоряет выполнение запросов.
